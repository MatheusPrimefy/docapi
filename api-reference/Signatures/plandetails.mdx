* Request type: **\[GET]**

```json
https://api.premepay.com/v1/stores/:id/plans/:id
```


Through the above route you will receive a registered subscription plan.
Check out the possible responses from the Primefy API to this request below.

### Response

- 200: OK
    
    This is the answer for cases where your request is successful.
    
    ```json
    {
        "id": 19888,
        "storeId": 359,
        "name": "Plano Mensal",
        "description": "Descrição Plano Mensal",
        "frequency": 2,
        "productCategory": null,
        "products": null,
        "maxCharges": 12,
        "chargeDaysBefore": 1,
        "paymentTypes": [
            0,
            1,
            3
        ],
        "amount": 19.80,
        "currency": "BRL",
        "gracePeriod": 0,
        "status": 0,
        "tax": null,
        "isVisible": true
    }
    ```
    
- 401: Unauthorized
    
    This is the response for cases where the request was unsuccessful because it lacks valid authentication credentials for the target resource.
    If needed, you can review our [authentication process](https://www.notion.so/Authentication-c941a0cb8e2647b187d87fcf8a314b37?pvs=21).****
    
    ```json
    
    {
        "type": "https://httpstatuses.com/403",
        "title": "Unauthorized",
        "status": 403,
        "detail": null,
        "instance": null,
        "extensions": {}
    }
    ```
    
- 403: Forbidden
    
    This is the response for cases where the request has been banned. This happens when the server can understand the request request, but does not authorize the issue of an approval response.
    
    ```json
    {
        "type": "https://httpstatuses.com/403",
        "title": "Forbidden",
        "status": 403,
        "detail": null,
        "instance": null,
        "extensions": {}
    }
    ```
